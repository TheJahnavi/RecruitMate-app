Smart Hiring Web App

Objective: Build a professional, multi-role login, smart hiring web application using Replit. The application must be **mobile-screen friendly**, with a professional, high-end aesthetic that adheres to all UI/UX principles. The UI must feature a single, fixed navigation bar at the top, a light and dark mode toggle, and a clean, modern design based on the provided reference images. All data flows and logic must strictly follow the provided flowcharts, using a specific color palette.

---
1. Database Structure & Dummy Data
    The application will be built on the following schema:
    
    Users Table: id, name, email, password_hash, role (string: Super Admin, Company Admin, HR), company_id (FK), account_status, profile_pic_url.
    
    Companies Table: id, company_name, logo_url.
    
    Jobs Table: id, job_title, added_by_user_id (FK), hr_handling_user_id (FK), job_description, skills (array of strings), experience, note, positions_count, job_status, company_id.
    
    Candidates Table: id, candidate_name, email, job_id (FK), candidate_skills, candidate_experience, match_percentage, resume_url, hr_handling_user_id (FK), status, report_link, interview_link.
    
    Notifications Table: id, user_id (FK), message, timestamp, read_status.
    
    ToDos Table: id, user_id (FK), task, is_completed.
    
    Initial Data: Populate the Users, Jobs, and Candidates tables with 3 dummy records each.

        UI: A separate panel located in the right-most quarter of the page. This panel should be visually distinct from the main To-Do section.

        Notifications where read_status is True should appear with a standard font and background color.

3. AI Functionalities
Resume Analysis (on /hr/upload page):
    
    Agent 1: Data Extraction: An AI agent extracts candidate_name, email, candidate_skills, and candidate_experience from each uploaded resume.
    
    Agent 2: Percentage Match: Another AI agent compares the extracted skills and experience against the skills and experience fields of the selected job from the Jobs table. It returns a match_percentage score.
    
    Process: The results of both AI agents are combined and displayed in the results table, which can be sorted by match_percentage."
