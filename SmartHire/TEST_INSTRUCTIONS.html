SmartHire Job Functionality Test Instructions

Follow these steps to test the job management functionality in SmartHire:

Prerequisites

Ensure both servers are running:

  • Backend server on http://localhost:3001
  • Frontend server on http://localhost:5173

Test 1: Accessing the Jobs Page

1. Click the button below to open the application:

Open Jobs Page

2. If prompted, log in with one of the demo HR accounts:

  • Email: hr1@techcorp.com Password: password
  • Email: hr2@techcorp.com Password: password
  • Email: hr3@techcorp.com Password: password

3. Verify you can see the Job Management dashboard with stats cards and job table

Test 2: Adding a New Job

1. Click the "Add New Job" button in the top right corner

2. Fill in the job details:

  • Job Title: Frontend Developer
  • Job Description: Develop responsive web applications
  • Experience Required: 2-4 years
  • Skills: Add "React", "JavaScript", "CSS"
  • Number of Positions: 2
  • Status: Active
  • Additional Notes: Urgent hiring

3. Click "Create Job"

4. Verify a success message appears and the new job shows in the table

Test 3: Editing a Job

1. Find the "Frontend Developer" job in the table

2. Click the Edit (pencil) icon for that job

3. Verify the modal opens with pre-filled data

4. Make these changes:

  • Change Job Title to: Senior Frontend Developer
  • Change Experience Required to: 4-6 years
  • Add a new skill: TypeScript

5. Click "Update Job"

6. Verify a success message appears and the job details are updated

Test 4: Deleting a Job

1. Find the "Senior Frontend Developer" job in the table

2. Click the Delete (trash) icon for that job

3. Confirm deletion in the prompt

4. Verify a success message appears and the job is removed from the table

Expected Results

If all tests pass, the job functionality is working correctly:

  • Add New Job creates records in the database
  • Edit Job pre-fills data correctly and updates records
  • No authentication redirects to "/api/login" occur
  • All operations show appropriate success/error messages

Troubleshooting

If you encounter issues:

  • Check browser console for errors (F12)
  • Verify both servers are running on correct ports
  • Check that authentication middleware is applied to routes
  • Ensure API base URL is configured correctly
